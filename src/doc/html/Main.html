bigMapPic :: EstadoGloss -> IO Picture #

Esta é a função onde ocorrem as ações todas! |

na bigMapPic apenas será explicada a função quando está em modoJogo pois ao explicar essa, todas as outras são bastante explicitas pois se aplicam de forma semelhante, apenas com as imagens diferentes e coordenadas diferentes.

bigMapPic (ModoJogo jogo1, (x,y),playerR, playerL, playerRC, playerLC, menu1, menu2, menu3, menu4, niveis1, niveis2, niveis3, niveisVoltar, instrucoes, creditos, imgs1, imgs2, b)  
    | mod milisseconds 200 < 100 = return $ Translate (x-60) (y+80) $ Scale 0.2 0.2 $ Pictures $ (desenhaMapa (getMap jogo1, imgs1) (0,0)) ++ (getPlayer jogo1)
    | otherwise = return $ Translate (x-60) (y+80) $ Scale 0.2 0.2 $ Pictures $ (desenhaMapa (getMap jogo1, imgs2) (0,0)) ++(getPlayer jogo1)
    where milisseconds = round (b*50)
          getMap (Jogo mapa (Jogador (x,y) w bool)) = mapa
          getPlayer (Jogo mapa (Jogador (x,y) Oeste False)) = desenhaPlayer ([Boneco],(x,y),playerL)
          getPlayer (Jogo mapa (Jogador (x,y) Este False)) = desenhaPlayer ([Boneco],(x,y),playerR)
          getPlayer (Jogo mapa (Jogador (x,y) Oeste True)) = desenhaPlayer ([Caixinha,Boneco],(x,y-1),playerLC)
          getPlayer (Jogo mapa (Jogador (x,y) Este True)) = desenhaPlayer ([Caixinha,Boneco],(x,y-1),playerRC)

Esta função vai receber sempre o conjunto de argumentos do tipo EstadoGloss e devolver uma Picture, ou seja, uma imagem do mapa, portanto acaba por ter muita da sua função na estadoInicial.

Neste caso, bigMapPic vai receber o ModoJogo no seu estadoGloss, podendo assim dar load a qualquer mapa que lhe seja atriubuido.

  • 'mod milisseconds' é a parte da função responsável pela alteração do lado para o qual a princesa está virada, ou seja, a cada vez que o resto da divisao entre o tempo e 200 for menor que 100, a função irá alterar o estado da Imagem.
  • funções como Translate, Scale e assim servem, neste caso, apenas para ajustar as imagens e ficar bem desenhado.
  • getMap serve somente para devolver o mapa e o mesmo poder ser desenhado em separado do jogador
  • getPlayer é responsável por ir buscar o Jogador para o poder desenhar, de forma separada do Mapa. Também é esta que determina se vai ser desenhada, ou não, a caixa e o lado para o qual está virado, indo buscar a lista de imagens necessária para tal!

desenhaMapa ((h:t), imgs) (x,y) = (Translate (-1500) (fromIntegral (y*(-160))) (Pictures (desenhaLinha (h,imgs) x))):(desenhaMapa (t,imgs) (x,y+1))

Nesta função, vamos recorrer à função desenhaLinha para a utilizar recursivamente até todas as linhas estarem completas. Assim, usamos o translate com alteração dos y a multiplicar por (-160) para criar o espaço certo na vertical de acordo com o tamanho das imagens!

desenhaLinha :: ([Peca], [Picture]) -> Int -> [Picture] #

desenhaLinha :: ([Peca],[Picture]) -> Int -> [Picture]

Tal como no y da função anterior, a multiplicação de x por 160 funciona justamente para ajustar o tamanho das imagens ao mapa, isso tem de ser sempre repetido recursivamente para ir desenhando até ao fim da lista.

pecas :: (Peca, [Picture]) -> Picture #

pecas :: (Peca, [Picture]) -> Picture

pecas (Caixa, imgs) = imgs !! 1

pecas (Vazio, imgs) = Blank

Esta função vai buscar à lista das imagens atribuida acima, a imagem correspondente a cada tipo de Peca.

Blank devolve uma imagem "vazia", ou seja, não substitui por nada. Podemos inventar algo para preencher o fundo dos Vazio, mas não acho que seja o melhor.

reageEvento :: Event -> EstadoGloss -> IO EstadoGloss #

Esta função é responsável por alterar a interface do jogo e a posição do jogador ao pressionar determinadas teclas do teclado.

Menu Principal

Quando no menu principal estas funções são responsáveis por alternar entre as opções Jogar, Sair, Instruções e Créditos ao clicar nas setas pra cima e para baixo. Esta função também é responsável por direcionar o jogador para o menu dos níveis, das instruções e dos créditos ao clicar no enter.

Jogar

Quando selecionada a opção Jogar:

  • Ao clicar na seta para cima o menu vai permanecer igual.
  • Ao clicar na seta para baixo a opção selecionada vai passar a ser a opção Instruções.
  • Ao clicar no enter o jogador vai ser direcionado para outro menu com os níveis disponíveis para jogar.

Ao selecionar qualquer outra tecla não ocorrerá nada. Uma exceção é a tecla esc que irá fechar o jogo quando no menu principal.

Sair

Quando selecionada a opção Sair:

  • Ao clicar na seta para cima a opção selecionada vai passar a ser a opção Créditos.
  • Ao clicar na seta para baixo o menu vai permanecer igual.
  • Ao clicar no enter o jogador vai fechar o jogo.

Ao selecionar qualquer outra tecla não ocorrerá nada. Uma exceção é a tecla esc que irá fechar o jogo quando no menu principal.

    exitSuccess

Créditos

Quando selecionada a opção Créditos:

  • Ao clicar na seta para cima a opção selecionada vai passar a ser a opção Instruções.
  • Ao clicar na seta para baixo a opção selecionada vai passar a ser a opção Sair.
